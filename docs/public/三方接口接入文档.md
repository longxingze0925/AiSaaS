# AiSaaS 三方接口接入文档

本文只面向外部业务平台 / SaaS 后端。浏览器、移动端和前端页面不要保存 AiSaaS Server Key，也不要直接请求 AiSaaS Server API。

## 1. 接入架构

```text
浏览器 / App
  -> 你的业务后端
    -> AiSaaS Server API
      -> 第三方 AI 平台
      -> AiSaaS 素材缓存
```

AiSaaS 负责客户状态、订阅、余额、预扣、扣费、失败释放、素材缓存和审计。你的业务后端负责自己的登录态、业务权限和页面交互。

## 2. 接入准备

在 AiSaaS 管理后台完成这些配置：

1. 创建应用。
2. 创建 Server Key，scope 至少包含 `ai:invoke`。
3. 创建客户，并给客户开通订阅。
4. 给客户余额账户充值，确认 `ai_enabled=true`。
5. 配置 AI 渠道、模型商品和模型路由。

Server Key 明文只显示一次，必须保存在业务后端密钥管理中。

## 3. 认证和通用规则

通用认证：

```http
Authorization: Bearer aissk_...
```

低层 AI 接口必须传本次实际消费客户：

```http
X-AiSaaS-Customer-Id: <customer_id>
```

幂等请求建议传：

```http
Idempotency-Key: <unique-key>
```

AI 调用成功或失败时，响应头会尽量返回：

```http
x-aisaas-usage-id: <usage_id>
```

标准业务接口响应包裹：

```json
{
  "code": 0,
  "message": "ok",
  "data": {},
  "request_id": "req_xxx"
}
```

低层 OpenAI 兼容 AI 接口返回三方兼容 JSON，不一定使用上面的包裹结构。

### 3.1 素材 URL 访问规则

业务后端可以调用 AiSaaS Server API，但浏览器和 App 不应该携带 Server Key 访问这些接口。用于前端展示的素材 URL 需要按下面规则处理：

| 字段 | 推荐用途 | 浏览器访问规则 |
| --- | --- | --- |
| `asset.url` | 素材展示 / 播放 / 下载 | 不需要 Server Key |
| `asset.public_url` | 素材展示 / 播放 / 下载 | 不需要 Server Key |
| `asset.thumbnailUrl` | 缩略图展示 | 不需要 Server Key |
| `job.assets[].url` | 生成结果展示 | 不需要 Server Key |
| `work.primary_asset_url` | 作品主素材展示 | 不需要 Server Key |
| `work.cover_asset_url` | 作品封面展示 | 不需要 Server Key |

这些字段在生产接入中推荐返回**浏览器可直接访问**的公开 CDN URL、对象存储公开 URL 或短期签名 URL，不需要 `Authorization`，不需要 Server Key，也不需要上传 token。

如果当前部署返回的是 `/api/server/web/v1/assets/{id}/download` 这类 Server API 地址，并且该地址要求 `Authorization: Bearer aissk_...`，影织前端不能直接访问，必须由影织后端代理下载/转发。这个模式只作为兼容方案；正式生产推荐配置对象存储/CDN/短期签名 URL，让图片、视频、音频可以直接在浏览器加载。

## 4. 低层 AI 网关接口

这些接口适合业务后端直接用 OpenAI 兼容方式接入。

| 方法 | 路径 | 用途 |
| --- | --- | --- |
| `GET` | `/api/server/ai/v1/models` | 查询当前客户可用模型商品 |
| `POST` | `/api/server/ai/v1/chat/completions` | 文本对话 |
| `POST` | `/api/server/ai/v1/embeddings` | Embedding |
| `POST` | `/api/server/ai/v1/images/generations` | 同步图片生成 |
| `POST` | `/api/server/ai/v1/videos/generations` | 同步视频生成 |
| `POST` | `/api/server/ai/v1/images/jobs` | 异步图片生成任务 |
| `POST` | `/api/server/ai/v1/videos/jobs` | 异步视频生成任务 |
| `POST` | `/api/server/ai/v1/audio/jobs` | 异步音频生成任务 |
| `GET` | `/api/server/ai/v1/jobs/{id}` | 查询异步任务 |

示例：

```bash
curl https://aisaas.example.com/api/server/ai/v1/chat/completions \
  -H "Authorization: Bearer aissk_..." \
  -H "X-AiSaaS-Customer-Id: 00000000-0000-0000-0000-000000000001" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-text-basic",
    "messages": [{"role": "user", "content": "hello"}],
    "max_tokens": 256
  }'
```

## 5. Web 统一 AI 任务接口

推荐 Web 产品优先使用统一任务接口，由业务后端传 `customer_id`，AiSaaS 负责预扣、提交、轮询、缓存和结算。

| 方法 | 路径 | 用途 |
| --- | --- | --- |
| `GET` | `/api/server/web/v1/ai/models` | 查询模型商品和能力 |
| `POST` | `/api/server/web/v1/ai/jobs` | 创建图片 / 视频 / 音频任务 |
| `GET` | `/api/server/web/v1/ai/jobs` | 分页查询客户任务 |
| `GET` | `/api/server/web/v1/ai/jobs/{id}` | 查询任务详情 |
| `POST` | `/api/server/web/v1/ai/jobs/{id}/cancel` | 取消本地任务并释放未扣费预扣 |
| `POST` | `/api/server/web/v1/ai/jobs/{id}/retry` | 重新查询第三方任务 |

### 5.1 模型列表返回结构

`GET /api/server/web/v1/ai/models` 返回 OpenAI 风格列表，不使用标准 `ApiResponse` 包裹。

可选查询参数：

| 参数 | 说明 |
| --- | --- |
| `type` | `text` / `image` / `video` / `audio` / `embedding` / `multimodal` |
| `customer_id` | 可选。传入后会校验客户状态和当前应用订阅，再返回模型 |

客户标识也可以通过请求头传递：

```http
X-AiSaaS-Customer-Id: <customer_id>
```

兼容旧 header：

```http
X-EntitleHub-Customer-Id: <customer_id>
```

### 5.2 创建任务请求

`POST /api/server/web/v1/ai/jobs` 使用顶层字段。`type` 是推荐字段；为兼容旧接入，也支持 `job_type` 和嵌套 `payload`，但新接入不要依赖嵌套写法。

```json
{
  "customer_id": "00000000-0000-0000-0000-000000000001",
  "type": "video",
  "model": "video-pro",
  "prompt": "a clean product video",
  "ratio": "16:9",
  "aspectRatio": "16:9",
  "resolution": "720p",
  "duration": 5,
  "durationSec": 5,
  "n": 1,
  "sourceMode": "image",
  "inputMode": "image",
  "referenceAssets": [
    {
      "assetId": "10000000-0000-0000-0000-000000000001",
      "kind": "image",
      "role": "reference"
    }
  ],
  "referenceAssetIds": ["10000000-0000-0000-0000-000000000001"],
  "firstFrameAssetId": "10000000-0000-0000-0000-000000000002",
  "lastFrameAssetId": "10000000-0000-0000-0000-000000000003"
}
```

字段说明：

| 字段 | 说明 |
| --- | --- |
| `customer_id` | 必填，AiSaaS 客户 ID |
| `type` | 必填，`image` / `video` / `audio` |
| `model` | 必填，模型商品 ID，对应模型列表里的 `id` |
| `prompt` | 生成提示词 |
| `ratio` / `aspectRatio` | 比例，二者等价 |
| `resolution` | 分辨率或清晰度 |
| `duration` / `durationSec` / `durationSeconds` | 时长秒数，三者等价 |
| `n` | 图片数量 |
| `inputMode` / `sourceMode` | 输入模式，二者等价，建议使用 `inputMode` |
| `referenceAssets` | 结构化参考素材，支持 `assetId`、`kind`、`role` |
| `referenceAssetIds` | 普通参考素材 ID 列表 |
| `firstFrameAssetId` | 首帧图片素材 ID |
| `lastFrameAssetId` | 尾帧图片素材 ID |

`referenceAssets[].role` 支持：

| role | 说明 |
| --- | --- |
| `reference` | 普通参考素材 |
| `first_frame` | 首帧素材 |
| `last_frame` | 尾帧素材 |

素材数量、素材类型、MIME、文件大小、参考视频/音频时长都会按模型 `capabilities` 校验。

影织前端读取规则：

- 成功后优先读取 `job.assets[]` 展示结果素材。
- 作品详情和发布、收藏、下载使用 `job.workId`。
- 扣费展示使用 `chargedMinor`，预扣展示使用 `heldMinor`。
- 失败原因展示使用 `failureReason`。
- `asset.url` 是 AiSaaS 缓存后的地址，不要直接使用第三方原始 URL。

任务状态：

| 状态 | 含义 |
| --- | --- |
| `pending` / `submitted` | 已创建或已提交 |
| `running` | 第三方处理中 |
| `caching` | 第三方成功，AiSaaS 正在缓存素材 |
| `succeeded` | 成功并完成扣费 |
| `failed` / `provider_failed` | 失败并释放或退款 |
| `cancelled` | 本地取消 |
| `timeout_review` | 超时进入人工复核 |

## 6. 客户信息接口

这些接口用于业务后端把自己的用户映射到 AiSaaS 客户。

| 方法 | 路径 | 用途 |
| --- | --- | --- |
| `POST` | `/api/server/web/v1/customers/register` | 注册 AiSaaS 客户 |
| `POST` | `/api/server/web/v1/customers/login` | 校验客户账号密码 |
| `GET` | `/api/server/web/v1/customers/{id}` | 查询客户资料 |
| `GET` | `/api/server/web/v1/customers/{id}/balance` | 查询客户余额 |
| `GET` | `/api/server/web/v1/customers/{id}/usage` | 查询客户 AI 用量 |
| `GET` | `/api/server/web/v1/customers/{id}/plan` | 查询客户订阅 |

客户登录接口只用于校验 AiSaaS 客户账号密码；浏览器 session 仍由你的业务后端自己维护。

### 6.1 注册 / 登录请求体

`POST /api/server/web/v1/customers/register` 请求体：

```json
{
  "email": "user@example.com",
  "password": "StrongPassword123!",
  "name": "用户昵称"
}
```

字段规则：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `email` | 是 | 客户邮箱，AiSaaS 内唯一 |
| `password` | 是 | 客户密码，按 AiSaaS 密码强度校验 |
| `name` | 否 | 用户昵称，空字符串会按未传处理 |

`phone`、`invite_code` 当前不是正式契约字段，不要在影织侧依赖。后续如果要手机号登录或邀请码注册，应单独扩展接口契约。

`POST /api/server/web/v1/customers/login` 请求体：

```json
{
  "email": "user@example.com",
  "password": "StrongPassword123!"
}
```

## 7. 素材库接口

上传文件推荐流程：

1. 业务后端用 Server Key 创建上传会话。

### 7.4 上传 token 安全规则

`POST /api/server/web/v1/assets/upload-url` 创建一次性上传会话，请求示例：

```json
{
  "customer_id": "00000000-0000-0000-0000-000000000001",
  "folder_id": "50000000-0000-0000-0000-000000000001",
  "file_name": "reference.png",
  "kind": "image",
  "asset_role": "reference",
  "mime_type": "image/png",
  "file_size": 1024000,
  "metadata": {}
}
```

响应示例：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "upload": {
      "upload_id": "60000000-0000-0000-0000-000000000001",
      "method": "PUT",

}
```

- `upload_token` 单次有效，上传成功后立即消费，重复上传会返回冲突错误。

- 如果创建会话时传了 `file_size`，实际上传字节数必须一致。

### 8.3 作品动作请求和响应

收藏、取消收藏、下载、发布、取消发布、删除都需要传当前客户 ID：

```json
{
  "customer_id": "00000000-0000-0000-0000-000000000001"
}
```

发布作品可以额外传标签：

```json
{
  "customer_id": "00000000-0000-0000-0000-000000000001",
  "tags": ["精选", "视频"]
}
```

下面示例里的 `work` 只展示动作相关关键字段；真实响应会返回完整作品对象，字段结构同作品列表里的单个 `work`。

`POST /api/server/web/v1/works/{id}/favorite` 和 `DELETE /api/server/web/v1/works/{id}/favorite` 返回：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "work": {
      "id": "40000000-0000-0000-0000-000000000001",
      "title": "AI 生成视频",
      "work_type": "video",
      "visibility": "private",
      "favorited": true
    },
    "work_id": "40000000-0000-0000-0000-000000000001",
    "customer_id": "00000000-0000-0000-0000-000000000001",
    "favorited": true,
    "favoritedAt": "2026-07-09T12:00:00Z"
  },
  "request_id": "req_xxx"
}
```

取消收藏时 `favorited=false`，`favoritedAt=null`。影织推荐以 `data.work` 刷新当前作品卡片，同时用 `data.favorited` 更新按钮状态。

`POST /api/server/web/v1/works/{id}/publish` 和 `POST /api/server/web/v1/works/{id}/unpublish` 返回：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "work": {
      "id": "40000000-0000-0000-0000-000000000001",
      "title": "AI 生成视频",
      "work_type": "video",
      "visibility": "public",
      "publishedAt": "2026-07-09T12:00:00Z"
    }
  },
  "request_id": "req_xxx"
}
```

取消发布后 `visibility=private`，`publishedAt` 可能保留历史发布时间或按状态展示为不可见，影织前端应以 `visibility` 判断是否在画廊展示。

`POST /api/server/web/v1/works/{id}/download` 返回：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "work": {
      "id": "40000000-0000-0000-0000-000000000001",
      "title": "AI 生成视频",
      "work_type": "video"
    },

  "request_id": "req_xxx"

```

影织下载时读取 `data.downloadUrl`。如果为空，表示该作品当前没有可下载主素材，应提示用户稍后重试或联系平台。

`DELETE /api/server/web/v1/works/{id}` 返回：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "deleted": true,
    "work_id": "40000000-0000-0000-0000-000000000001"
  },
  "request_id": "req_xxx"
}
```

### 8.4 公开画廊筛选参数

`GET /api/server/web/v1/gallery` 支持：

| 参数 | 说明 |
| --- | --- |
| `type` | `image` / `video` / `audio` / `file` |
| `customer_id` | 可选，传入后会标记当前客户是否收藏 |
| `page` | 页码，从 1 开始 |
| `page_size` | 每页数量，最大 100 |
| `sort` | `latest` / `hot` / `featured` |

排序规则：

- `latest`：按发布时间倒序。
- `hot`：优先收藏数，再按精选分和发布时间排序。
- `featured`：按后台精选分和发布时间排序，也是默认排序。

## 9. 常见错误

标准错误响应：

```json
{
  "code": 422,
  "message": "validation_failed",
  "error_code": "MODEL_CAPABILITY_UNSUPPORTED",
  "request_id": "req_xxx",
  "details": {
    "model": "video-pro",
    "field": "durationSec"
  }
}
```

字段说明：

| 字段 | 说明 |
| --- | --- |
| `code` | HTTP 状态码或平台错误码 |
| `message` | 粗粒度错误类型，适合做默认提示 |
| `error_code` | 精确业务错误码，适合影织后端做用户提示映射 |
| `request_id` | 排查请求链路使用，反馈问题时必须保留 |
| `details` | 可选字段级上下文；可能为空对象 |

| HTTP | message | 处理建议 |
| --- | --- | --- |
| `401` | `unauthenticated` | 检查 Server Key 是否缺失或格式错误 |
| `403` | `forbidden` / `subscription_inactive` | 检查 scope、客户状态、订阅 |
| `404` | `not_found` | 检查客户、任务、素材或模型代码 |
| `409` | `conflict` | 检查重复提交或状态冲突 |
| `422` | `validation_failed` | 检查请求字段、模型能力、素材限制 |
| `429` | `rate_limited` | 降低请求频率或联系平台调整限流 |
| `502` / `503` | provider 或依赖异常 | 记录 `request_id` / `x-aisaas-usage-id` 后重试或排查三方渠道 |

## 10. 上线检查

- Server Key 只放业务后端，不进前端和公开仓库。
- 每次 AI 消费都传正确 `customer_id`。
- 业务后端保存 AiSaaS `job.id` 和 `x-aisaas-usage-id`，不要只保存三方任务 ID。
- 前端展示的素材 URL 使用 AiSaaS 返回的 URL，不直接使用第三方原始 URL。
- 生产环境使用第三方可访问的对象存储公开 URL 或短期签名 URL。
- 接入方自己的业务入口也要做限流，不能只依赖 AiSaaS 平台限流。
