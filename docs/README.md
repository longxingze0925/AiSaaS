# 文档目录

本文说明 AiSaaS 文档分层，避免对外接入资料、内部开发资料和历史兼容资料混在一起。

## public

`docs/public/` 只放可以给外部业务平台或 SaaS 后端看的资料。

- `三方接口接入文档.md`：三方业务后端对接 AiSaaS Server API 的唯一入口文档。

## internal

`docs/internal/` 只给项目开发、维护、部署人员使用。

- `README.md`：内部文档入口。
- `development/`：开发规则、任务拆分、路线图、产品魔改和文档检查。
- `architecture/`：系统架构、数据库、权限错误码、完整内部 API 字典。
- `operations/`：部署、运维、安全测试和上线检查。

## archive

`docs/archive/` 放非主线或历史兼容资料。这里的内容不作为新功能开发入口，只用于维护旧兼容能力时参考。

- `README.md`：历史兼容资料说明。

## 根目录约束

根目录只保留项目入口、运行配置和工程目录。新增文档默认放入 `docs/` 对应分层，不再直接堆到根目录。
